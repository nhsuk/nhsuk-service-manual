# Contributing to the Technical Review Group guidance

## Overview

The Technical Review Group (TRG) pages are here to help teams go through this key NHS England Architecture governance process. Contributions should help readers navigate to the key information quickly and answer common questions. 

## Review Process

1. **Submission**: Contributors must submit their changes in a pull request (PR) to `main`, but from a branch specific to the changes
3. **Review**: The maintainers will perform a detailed review, focusing on:
   - Simple and clear content - following the [NHS Service Manual content guide](https://service-manual.nhs.uk/content)
   - If relevant, code quality and adherence to coding standards
   - Alignment with NHSE architecture and design principles
 
## Criteria for Acceptance

Contributions will be accepted based on the following criteria:

- **Clarity**: Content must be simple and clear
- **Consistent**: Are there related pages that also need updating based on the desired changes?
- **Alignment**: Contributions must align with the NSHE architecture and design principles.

## Roles of Reviewers

- **Helpful**: Work with the contributor to improve and align
- **Process impact**: Spot where a change actually changes or contradicts the existing process. If there is a process impact, that content must be excluded from the current contribution. Then consider raising a new issue
- **Aligns with Service Manual**: The Service Manual is based on the Government Digital Service (GDS) standards to make content easier and clearer. Work through contributions and drive content to meet the standard
